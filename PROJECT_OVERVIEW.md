# Fear Insight Project Architecture & Overview

This document provides a comprehensive overview of the Fear Insight platform. It details the underlying technology stack, application architecture, feature logic, and the flow of core systems such as e-commerce checkout and admin portal capabilities.

---

## 💻 Tech Stack

### Core Technologies
- **Framework:** [Next.js 14](https://nextjs.org/) (App Directory / Router) 
- **Library:** [React 18](https://react.dev/)
- **Language:** TypeScript
- **Styling:** [Tailwind CSS](https://tailwindcss.com/) with `tailwindcss-animate`
- **UI Components:** [Radix UI](https://www.radix-ui.com/) (Accessible base components integrated via shadcn/ui patterns)

### Backend & Services
- **Database & Storage:** [Supabase](https://supabase.com/) (PostgreSQL wrapper + Storage capabilities)
- **Payments:** [Stripe API](https://stripe.com/) (`stripe` node module)
- **Emails:** [Nodemailer](https://nodemailer.com/)
- **Data Fetching:** Standard Next.js server fetch + React `cache()` + `@tanstack/react-query`

### Animation & Utilities
- **Animations:** Framer Motion
- **Form & Validation:** React Hook Form + Zod
- **Typography:** Custom local and Google Fonts (Geist Sans, Geist Mono, Montserrat)

---

## 🏗 Directory Structure & Domains

- `/app`: Next.js core application routes and layout files.
  - `/admin`: The secure admin portal.
  - `/api`: Backend API routes for checkout, webhooks, analytics, and admin actions.
  - `/context`: React Context providers (e.g., `CartContext.tsx`).
- `/components`: Reusable UI components, separated into domains like `admin`, `landing-page`, `product`, `ui`.
- `/functions`: Reusable async logic related to Supabase and business logic (e.g., `getAllProducts.ts`, `createOrder.ts`, `notifyNewOrderPayment.ts`).
- `/hooks`: Custom React hooks for specific responsibilities (e.g., `useAllProducts.ts`, `use-toast.ts`).
- `/lib` & `/utils`: Helper utility functions and structured data/SEO rules.

## 🔄 Core Application Flow 

### 1. Landing Page & Products Fetching
- **Data Lifecycle:** The application pulls items from the Supabase `products` table. Through Server Components, `getAllProducts` (in the `functions/` utility) connects directly to the Supabase PostgreSQL database.
- **Caching Mechanism:** React's `cache()` is utilized around data fetch operations inside `getAllProducts.ts` to deduplicate queries implicitly generated by different components rendering on the same server passage.

### 2. Cart Management & E-Commerce Flow
- **Global State Context:** `CartContext.tsx` handles cart additions, subtractions, subtotal aggregations, discounts, and shipping logic across the application. Local cart data is hydrated using `localStorage`.
- **Promo Codes:** Specific codes (e.g., `INFLUENCER10`, `SAVE10`) provide a flat 20% discount against the subtotal.
- **Handling Combinations:** Users can pick attributes such as **Color** and **Size**. Checks are performed against variant arrays before the item successfully mounts into the Cart state.

### 3. Payment & Checkout (Stripe Integration)
- **Session Instantiation:** When an order is ready to be created, a POST request triggers `/api/create-checkout-session/route.ts`.
- **Discount & Metadata Logic:** It proportionally calculates promo discounts against each product in the cart object alongside flat shipping / tax fees, constructing `Stripe.Checkout.SessionCreateParams.LineItem[]`.
- **Callback Loop:** 
  1. Frontend navigates to Stripe Checkout URL.
  2. Stripe acts upon validation and redirects the user back to `/checkout-success` or `/checkout-cancel` with their custom `order_id` tagged as metadata.
  3. A robust `/api/stripe-webhook/route.ts` reacts to backend events like `checkout.session.completed` from Stripe directly.
- **Email Dispatching:** The successful resolution uses Nodemailer to dispatch confirmation and status emails (via `functions/sendOrderConfirmationEmail.ts`).

### 4. Admin Portal & Permissions
- **Security Check Mechanism:** Restricted content relies on `middleware.ts`. A custom `admin_session` signed HTTP cookie guards access to any path starting with `/admin` except the login path itself.
- **Metrics Tracking:** Built-in `<VisitorTracker />` monitors visitor stats quietly in the background and commits hits to the Supabase analytics tables.
- **Media Optimization:** 
  - Submitting a new Product via the admin product table triggers an isolated `/api/admin/upload-product` operation. 
  - To prevent duplicates and URL loss, names follow an incremental sequence convention (`f001.jpg`, `f002.jpg`).
  - Storage is connected straight from Supabase Storage and arrays of their public URLs are injected directly into the `images` array column on the database row. Existing images are preserved smoothly by concatenating old URLs and new URLs transparently.

## ⚙️ Primary Logics Summary

- **Authentication Logic:** HTTP-only cookies managed via edge `middleware.ts`.
- **Pricing Logic:** `Total = (Subtotal - 20% Promo [if applicable]) + (Type-Specific Shipping [e.g., $12.99 Express] + Tax)`.
- **Email Triggering Logic:** Async triggers decoupled from frontend. Supabase function wrappers fetch `orderId` relationships, parse content, and distribute using Nodemailer payload configurations.
- **Database Logic:** Data layers live in `functions/`. Supabase JS is initialized safely. Queries execute securely natively utilizing `.select()`, `.insert()`, and `.update()` configurations connected strictly over environment secure keys `NEXT_PUBLIC_SUPABASE_ANON_KEY` / `SUPABASE_SERVICE_ROLE_KEY`.
